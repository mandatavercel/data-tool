POLICY OVERVIEW

📘 정책의 목적

본 정책은 글로벌 기관투자자 대상 데이터 상품을 제공하는 과정에서, 데이터 원천사(파트너)와의 수익배분율을 일관되고 설득력 있게 산정하기 위한 기준이다. 모든 신규·갱신 파트너십 계약에 본 정책이 표준 산정 기준으로 적용되며, 파트너에게 제시되는 배분율의 근거 문서로 활용된다.

본 정책은 두 가지 목표를 동시에 달성한다. 첫째, 데이터 원천사의 본질적 가치를 객관적 지표로 평가하여 정당한 보상을 보장한다. 둘째, 맨데이터의 상품화·세일즈·운영 기여도가 매출에 비례하여 적정하게 회수되도록 한다. 따라서 단순히 "데이터를 제공한다"는 사실이 아닌, 시장 대표성·데이터 가치·상업화 부담을 정량 평가한 결과로 배분율을 산정한다.

🏛 5대 산정 원칙

PRINCIPLE 01
정량 평가 우선
사전 정의된 7개 평가 항목의 가중 점수에 의해 산정. 모든 파트너에게 동일한 산정 공식 적용.
PRINCIPLE 02
가치-비용 균형
데이터 원천 가치(70%)와 상업화 비용 부담(30%)을 동등한 비중으로 반영.
PRINCIPLE 03
Tier-up 연동
연간 순매출 규모에 따라 자동 상향되는 단계형 구조. 파트너에게 매출 성장 인센티브 제공.
PRINCIPLE 04
상하한 캡
파트너 배분율은 25% 미만 또는 60% 초과로 산정되지 않음. 양측의 최소·최대 이익 보호.
PRINCIPLE 05
정책의 투명성
산정 공식, 평가 항목, 가중치, Tier 구조 모두 공개. 산정 근거 자료 일체 제공.

📐 산정 공식

STEP 1 · 가중평균 점수 Σ ( 평가 항목 점수 × 가중치 ) → 1.00 ~ 5.00
STEP 2 · 조정폭 ( 가중평균 − 3.0 ) × 6%p → 최대 ±12%p
STEP 3 · 최종 파트너 배분율 MAX( 25% , MIN( 60% , 기본배분율 + 조정폭 + Tier 가산 ) )

기본 배분율은 데이터 원천사 유형별로 사전 설정된 출발점이며, 평가 점수에 따라 ±12%p 범위 내에서 조정된다. Tier 가산은 연간 순매출 구간에 따라 자동 적용(+0/+3/+5%p)되며, 최종 결과에 캡(25%~60%)이 적용된다.

🔒 캡(Cap) 정책

LOWER CAP
25%
데이터 원천사 없이는 사업이 성립하지 않는다는 점을 반영. 어떤 평가 결과에서도 원천사의 최소 가치를 보장하기 위한 안전선.
UPPER CAP
60%
맨데이터가 부담하는 상품화·세일즈·계약·운영 비용의 회수와 신규 투자 여력 확보를 위한 최소 마진선.
REFERENCE TABLE · ①

데이터 원천사 유형별 기본 배분율

모든 산정의 출발점. 데이터 유형이 글로벌 기관투자자 시장에서 일반적으로 갖는 본질적 가치·상품화 난이도·거래 사례를 종합 반영한 사전 설정 값이다.

REFERENCE TABLE · ②

평가 항목별 가중치 (총합 100%)

데이터 자체의 가치(시장 커버리지·점유율·독점성·활용성·깊이) 70% + 상업화 가능성(상품화·판매 용이성) 30%. 데이터 활용성이 단일 항목 최고 가중치(20%) — 글로벌 기관투자자가 데이터를 매입하는 본질적 이유이기 때문.

REFERENCE TABLE · ③

매출 구간별 Tier-up 가산

"순매출"은 환불·할인·플랫폼 수수료 차감 후 맨데이터에 인식되는 매출액. 매출 성장 시 배분율이 자동 상향되는 단계형 구조로, 파트너에게 명확한 성장 인센티브를 제공.

SCORING RUBRIC

📐 평가 항목별 1~5점 기준 정의

각 평가 항목에서 1점(매우 낮음) ~ 5점(매우 높음)을 매기는 기준. 평가자는 객관적 근거(시장조사, 매출 점유율 등)를 확보한 후 점수를 부여한다. 모든 항목은 점수가 높을수록 파트너 배분율이 상향되도록 정의된다.

공식 파트너 배분율 = MAX(25%, MIN(60%, 기본배분율 + (가중평균−3.0)×6%p + Tier 가산))  ·  MANDATA Internal Policy v1.0
