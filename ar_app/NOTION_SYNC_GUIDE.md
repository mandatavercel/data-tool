# 노션 정산 현황 동기화 설정 (공유용, 앱 → 노션 단방향)

정산 현황표 탭의 **🔄 노션으로 내보내기** 버튼으로, 앱(Neon)의 최신 데이터를
노션 DB **"AR 정산 현황 (자동 동기화)"** (Mandata_HQ › AR / AP)에 push 합니다.

설정은 한 번만 하면 됩니다 (약 3분).

## ① 노션 내부 통합(Integration) 만들기
1. https://www.notion.so/my-integrations → **New integration**.
2. 이름 예: `Mandata AR Sync`, 연결 워크스페이스 선택 → 저장.
3. **Internal Integration Secret** (`ntn_...` 또는 `secret_...`) 복사.

## ② 통합을 DB에 연결(공유)
1. 노션에서 **"AR 정산 현황 (자동 동기화)"** DB 페이지 열기.
2. 우측 상단 `···` → **Connections(연결) → 통합 추가** → 위에서 만든 `Mandata AR Sync` 선택.
   (이걸 해야 통합이 그 DB를 읽고 쓸 수 있습니다.)

## ③ Streamlit secrets에 토큰 입력
배포된 앱 → **Settings → Secrets** 에 추가 후 저장(앱 자동 재시작):
```toml
NOTION_TOKEN = "ntn_여기에_통합_시크릿"
# (선택) 다른 DB를 쓰려면:
# NOTION_DATABASE_ID = "747e0897-e2ba-4c6d-8333-c1a02da73ea2"
```
DB ID는 생략하면 현재 DB(`747e0897…`)가 기본 사용됩니다.

## ④ 사용
- 정산 현황표 탭의 **🔄 노션으로 내보내기** 클릭 → 신규/갱신/보관 건수 표시.
- 동작: 같은 (연도·배분사·계약) 행은 **갱신**, 없는 행은 **신규 생성**,
  더 이상 해당 없는 행은 **보관(archive)**. 노션에서 직접 고친 값은 다음 동기화 때 덮어쓰입니다.

## 메모
- 단방향(앱 → 노션) 공유용입니다. 입력/수정은 앱에서 하세요.
- 토큰 미설정 시 버튼 대신 안내문이 표시되며, 앱 동작에는 영향 없습니다.
